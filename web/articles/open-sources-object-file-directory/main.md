# 开源项目的文件目录

> 从 github 上找那些 star 最多最有名气的项目，分析他们的目录结构找共同点

## 基础的目录

    ${root}
        ├── build 构建、测试等各种脚本
        ├── dist 全称 distribution，存放发布后的代码
        ├── test 测试相关的目录
        ├── .gitignore
        ├── package.json
        ├── LICENSE
        └── README.md

dist 有的项目也用 bin 代替，bin 的全程 binary，一些需要编译的语言(如C)多用 bin。
